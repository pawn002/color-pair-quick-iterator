Notes

LCH Limits: This feature displays a collection of different buttons showing possible variants of your selected color limited by the sRGB color gamut. The visualization is built using a table element to provide an equitable experience in choosing colors based on lightness and chroma.

Object: A readout that shows two different values, a number in pixels that the current contrast level supports, and 'invisible' when the current contrast level is too low to support any object.

Perceptual contrast: Score that better gauges contrast than WCAG 2 in the creation of inclusive visual designs. For more information on the value used for Perceptual Contrast, refer to Myndex article Why APCA?

WCAG 2 compatible: A score compatible with use for WCAG 2.x success criteria. This score meets or exceeds WCAG 2 scores to better include those with atypical vision.

WCAG 2: Scores that are included for reference only—do not use this score for judging Accessible Contrast.

Display WCAG 2 compatible value: Option that swaps out perceptual contrast score visually displayed at top of the app. Screen reader users can access the same value regardless of this option in the Differences table located in the Color Metadata section.

Constant chroma: Option that forces the color sliders to limit light and dark variation of colors to tints and shades that have the same chroma and hue of the inputted color maintaining the objective identity of your input color.
